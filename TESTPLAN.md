# แผนการทดสอบ (Test Plan)
## ระบบเบิก-จ่ายอุปกรณ์ไอที (IT Equipment Checkout System)
### วิชา 21901-2003 การพัฒนาเว็บแอปพลิเคชัน

---

## 1. ขอบเขตการทดสอบ (Scope)

ทดสอบการทำงานของ Frontend ที่เชื่อมต่อกับ Backend API ครอบคลุม:
- ระบบ Login / Logout
- การแสดงรายการอุปกรณ์
- การเพิ่ม / ลบอุปกรณ์ (Admin)
- การตรวจสอบสิทธิ์ (Protected Route)
- การตรวจสอบข้อมูลจากฟอร์ม (Form Validation)

---

## 2. ทรัพยากร (Resources)

| บทบาท | จำนวน | หน้าที่ |
|---|---|---|
| ผู้พัฒนา / ผู้ทดสอบ | 1-2 คน | เขียน test case และทดสอบจริง |
| อาจารย์ที่ปรึกษา | 1 คน | ตรวจสอบผลการทดสอบ |
| เครื่องคอมพิวเตอร์ | 1 เครื่อง | รัน frontend + backend พร้อมกัน |

---

## 3. ช่วงเวลา (Timeline)

| สัปดาห์ | กิจกรรม |
|---|---|
| สัปดาห์ที่ 7 | เขียน test cases และทดสอบ unit (form validation) |

| สัปดาห์ที่ 8 (ปลาย) | วิเคราะห์ผลและสรุป go live / no go live |

## 5. เกณฑ์การตัดสิน (Go Live Criteria)

| เกณฑ์ | เป้าหมาย |
|---|---|
| Test cases ผ่าน | ≥ 90% (12/13 test cases) |
| ไม่มี critical bug | ระบบ login และแสดงข้อมูลต้องทำงานได้ 100% |
| Form validation ทำงานถูกต้อง | ผ่านทุก TC ที่เกี่ยวกับ validation |

---

## 6. ผลการวิเคราะห์ (Go Live / No Go Live)

| ผลทดสอบ | การตัดสินใจ |
|---|---|
| ผ่านเกณฑ์ทุกข้อ | **Go Live** — เปิดให้บริการได้ |
| ไม่ผ่าน critical criteria | **No Go Live** — แก้ไข bug ก่อน แล้วทดสอบซ้ำ |
